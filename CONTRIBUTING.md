# Welcome to Boring Tools Contributions

Thank you for considering contributing to Boring Tools! Your help makes this project better for everyone.

## Contribution Rules

To ensure consistency and quality, please follow these guidelines:

### Design & Implementation
- Follow the existing design system (Tailwind CSS classes, spacing, colors, animations, and theme)
- Maintain the same visual language as existing tools
- Do not modify unrelated files or break existing functionality
- New tools must solve real, everyday problems
- Avoid creating duplicate tools (check existing tools first)
- Keep code clean, modular, and well-organized
- Use meaningful variable and function names
- Ensure mobile responsiveness
- Reuse existing components and utilities where possible
- Avoid adding unnecessary heavy dependencies
- Update the README when adding new tools

### Tool Metadata
When adding a new tool, please include the following metadata in your tool's page:
- **Name**: Clear, descriptive name of the tool
- **Description**: Brief explanation of what the tool does
- **Category**: Appropriate category (e.g., Text, Utility, Media, Developer, etc.)
- **Keywords**: Relevant search terms for discoverability

## Pull Request Format

Please follow this format for your pull requests:

**Title:**
[Tool]: Tool Name

**Description:**

**Tool Name:** [Your Tool Name]
**Purpose:** [What problem does this tool solve?]
**Features:** [Key features and capabilities]
**Screenshots:** [Optional: Add screenshots demonstrating the tool]

### Before Submitting

Ask yourself:
- Does this tool save time, reduce effort, or solve a real problem?
- Is it consistent with the existing design and user experience?
- Have I tested it thoroughly across different screen sizes?
- Does it follow all the contribution rules above?

## Getting Started

1. Fork the repository
2. Create a new branch: `git checkout -b feature/your-tool-name`
3. Implement your tool following the existing patterns
4. Test your tool locally: `npm run dev`
5. Ensure the build works: `npm run build`
6. Commit your changes: `git commit -m "Add: [Tool Name]"`
7. Push to your branch: `git push origin feature/your-tool-name`
8. Open a Pull Request against the `main` branch

## Questions?

If you have any questions, feel free to open an issue or reach out to the maintainer.

Thank you for contributing to Boring Tools!
